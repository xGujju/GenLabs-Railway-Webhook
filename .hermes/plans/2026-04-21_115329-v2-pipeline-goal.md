# GenLabs LINE Slip Pipeline V2 Plan

> **For Hermes:** Execute this plan with TDD. Keep all work inside `xGujju/GenLabs-Railway-Webhook` only.

**Goal:** Turn the current tested webhook foundation into a Railway-ready V2 pipeline with a real OCR adapter boundary, runnable worker entrypoints, and a deployable single-cycle processing path from LINE media fetch to LINE reply.

Move actual processing into worker entrypoints that can be run on Railway as one-shot or looped jobs. **V2 scope now:**

**Out of scope for this slice:**

- object storage
- multi-tenant schema
- dashboard/frontend
